# 
            最初のセッション作成
          

          
            実際にセッションを作成して、tmuxの基本操作を学びます。
          

        


        
          
            ## 
              セッション作成の基本
            

            
              tmuxでセッションを作成する最も簡単な方法を紹介します。
            


            
              
                ### 
                  方法1: 名前を指定してセッション作成
                
                ```bash
$ tmux new-session -s myproject
# または短縮形
$ tmux new -s myproject
```
                
                  "myproject" という名前のセッションが作成され、すぐに接続されます。
                

              


              
                ### 
                  方法2: 名前なしでセッション作成
                
                ```bash
$ tmux new-session
# または短縮形
$ tmux new

# 結果: 0, 1, 2... という番号が自動割り当てされます
```
              


              
                ### 
                  方法3: セッション作成と同時にコマンド実行
                
                ```bash
$ tmux new-session -s work -c ~/projects/myapp
# セッションが作成され、~/projects/myapp ディレクトリで開始されます
```
              

            

          


          
            ## 
              セッション内での操作
            

            
              セッション内で使用する基本的なコマンドを紹介します。
            


            
              
                ### 
                  セッションから抜ける
                
                ```bash
# キーボードショートカット
Ctrl+B D

# または exit コマンド
$ exit
```
                
                  Ctrl+B D を使用すると、セッションは実行されたまま、ターミナルに戻ります。
                

              


              
                ### 
                  セッション一覧を確認
                
                ```bash
$ tmux list-sessions
# または短縮形
$ tmux ls

# 出力例:
# myproject: 1 windows (created Fri Feb 21 12:00:00 2026)
# work: 1 windows (created Fri Feb 21 12:05:00 2026)
```
              


              
                ### 
                  セッションに再接続
                
                ```bash
$ tmux attach-session -t myproject
# または短縮形
$ tmux a -t myproject
$ tmux attach -t myproject
```
              

            

          


          
            ## 
              実践例: AIコーディングセッション
            

            
              AIコーディングに最適化されたセッション設定の例を紹介します。
            


            
              ### 
                セッション作成例
              
              ```bash
# プロジェクトディレクトリでセッション作成
$ cd ~/projects/my-ai-project
$ tmux new-session -s dev -c .

# セッション内で複数のウィンドウを作成
Ctrl+B C  # 新しいウィンドウ作成
Ctrl+B C  # さらに新しいウィンドウ作成

# ウィンドウ1: エディタ（Claude Code CLI）
$ claude-code

# ウィンドウ2: テスト実行
$ npm test

# ウィンドウ3: ドキュメント参照
$ python -m http.server 8000
```
            


            :::message
**セッション作成完了**
セッションが作成できたら、次のページでウィンドウとペインの操作を学びます。
:::
          


          
            ## 
              トラブルシューティング
            

            
              
                ### 
                  問題: セッションが作成されない
                
                ```bash
# tmuxが正しくインストールされているか確認
$ which tmux
$ tmux -V

# tmuxサーバーを再起動
$ tmux kill-server
$ tmux new-session -s test
```
              


              
                ### 
                  問題: セッション名が重複している
                
                ```bash
# 既存のセッションを確認
$ tmux ls

# 同じ名前のセッションを削除
$ tmux kill-session -t myproject

# 新しいセッションを作成
$ tmux new-session -s myproject
```
              

            

          

        


        ### チャレンジ: tmux セッション管理コマンドを書いてみよう

新しいセッション作成、一覧表示、アタッチの基本コマンドを書いてください。

```tsx
# セッション管理の基本コマンド
# 1. 新しいセッションを "dev" という名前で作成

# 2. セッション一覧を表示

# 3. "dev" セッションにアタッチ

```

:::details 模範解答
```tsx
# セッション管理の基本コマンド
# 1. 新しいセッションを "dev" という名前で作成
tmux new-session -s dev

# 2. セッション一覧を表示
tmux list-sessions

# 3. "dev" セッションにアタッチ
tmux attach -t dev
```
:::
