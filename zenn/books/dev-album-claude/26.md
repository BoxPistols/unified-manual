# 
            tmux との統合 (AI Cockpit)
          

          
            エージェントの動作監視と実行環境の分離を、tmux のペイン管理で実現するワークフロー。

            ## 
              
              MCP (Model Context Protocol) 連携
            
            
              MCPサーバを事前に登録しておけば、Claude Code起動時に自動で接続されます。
            

            
              
                #### 
                  
                  事前設定で自動接続
                
                
                  `claude mcp add` で登録済みのサーバは、次回 `claude` 起動時に自動で利用可能になります。
                

              

              ```bash
# MCPサーバを事前登録（一度だけ実行）
$ claude mcp add --transport stdio github-server \
    -- npx -y @modelcontextprotocol/server-github

# 以降は claude を起動するだけでGitHub MCPが利用可能
$ claude
```

          ### チャレンジ: AI Cockpit のペインレイアウトを作ろう

tmux で Claude Code の AI Cockpit レイアウトを手動で構築するコマンドを書いてください。メインペインに Claude Code、右側にサーバーログとシェルを配置します。

```tsx
# AI Cockpit レイアウトの構築

# 1. セッションを作成

# 2. 左右にペインを分割（Claude Code | 監視エリア）

# 3. 右側ペインを上下に分割（ログ / シェル）

# 4. 左ペインで Claude Code を起動

# 5. 右上ペインで開発サーバーを起動
```

:::details 模範解答
```tsx
# AI Cockpit レイアウトの構築

# 1. セッションを作成
tmux new-session -s dev

# 2. 左右にペインを分割（Claude Code | 監視エリア）
tmux split-window -h

# 3. 右側ペインを上下に分割（ログ / シェル）
tmux split-window -v

# 4. 左ペインで Claude Code を起動
tmux select-pane -t 0
claude

# 5. 右上ペインで開発サーバーを起動
tmux select-pane -t 1
npm run dev
```
:::
