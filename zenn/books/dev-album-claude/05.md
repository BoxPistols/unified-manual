            スラッシュコマンド
          
          
            Claude CodeのREPL内で使用できるビルトインコマンドの全一覧。

            ## 
              
              コスト・コンテキスト監視
            
            
              
                `/cost`
                現在のセッションで消費した**トークン量と料金見積もり**を表示します。

              

              
                `/usage`
                **サブスクリプションプランの使用制限**とレート制限ステータスを表示します。

              

              
                `/context`
                現在のコンテキスト使用量を**カラーグリッド**で視覚的に表示します。

              

              
                `/stats`
                日次使用量、セッション履歴、モデル使用傾向をダッシュボード表示します。

              

            

            :::message
**/cost と /usage の違い**
/cost は現在のセッションのAPI消費量（トークン・金額）を表示し、/usage はサブスクリプションプラン全体の残り容量を確認するコマンドです。目的が異なるので混同に注意してください。
:::

            ## カスタムコマンド
            
              ビルトインコマンドに加え、独自のスラッシュコマンドを定義できます。
            

            
              
                #### プロジェクトコマンド
                `.claude/commands/` にMarkdownファイルを配置 → `/project:コマンド名` で呼び出し。チーム共有可能。

              

              
                #### ユーザーコマンド
                `~/.claude/commands/` にMarkdownファイルを配置 → `/user:コマンド名` で呼び出し。全プロジェクトで使用可能。

              

            

          


          
          
            ## クイック入力プレフィックス
            
              REPL内でプロンプトの先頭に特定の文字を入力することで、素早い操作が可能です。
            

            
              
                `/`
                コマンド / Skill呼び出し

              

              
                `!`
                Bashモード（直接シェル実行）

              

              
                `@`
                ファイルパス補完

              

            

          


          ### チャレンジ: カスタムコマンドを作成しよう

プロジェクトの .claude/commands/ ディレクトリに配置するカスタムスラッシュコマンドのMarkdownファイルを書いてください。コードレビューを実行するコマンドで、セキュリティとパフォーマンスの観点からレビューを行い、結果をマークダウンで出力するものです。

```tsx
# review.md - /project:review で呼び出されるコマンド

# ここにプロンプトを書いてください
```

:::details 模範解答
```tsx
# review.md - /project:review で呼び出されるコマンド

git diff HEAD~1 の変更をレビューしてください。

## レビュー観点
1. セキュリティ: 入力バリデーション、認証・認可の不備
2. パフォーマンス: N+1クエリ、不要な再レンダリング

## 出力形式
- 重要度（High/Medium/Low）
- 対象ファイルと行番号
- 問題の説明と修正案
```
:::
