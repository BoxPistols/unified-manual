# 
            プリフィックスキーの理解
          

          
            tmux操作の要となる「魔法の鍵」を使いこなしましょう。
          

        


        
          
            ## 
              プリフィックスキーとは？
            

            
              tmuxのコマンドを実行するために最初に押す特別なキーの組み合わせです。デフォルトでは **Control + B** に設定されています。
            


            
              ### 
                操作のリズム
              
              
                Control

                +

                B

                →

                （指を離す）

                →

                コマンドキー

              

              
                「CtrlとBを同時に押す」→「両方の指を離す」→「実行したいコマンドのキーを押す」という3ステップです。
              

            

          


          
            ## 
              なぜ必要なのか？
            

            
              
                ### コマンドの衝突を防ぐ
                
                  ターミナル上で動作する他のアプリケーション（vim, nano, sshなど）もショートカットキーを使用します。tmuxがこれらを奪ってしまわないように、明示的に「これからtmuxの命令を送るぞ」と宣言する必要があります。
                

              

              
                ### 誤操作の防止
                
                  ウィンドウを閉じたり、ペインを破棄したりする強力な操作を、誤って実行してしまうのを防ぎます。

            ## 
              練習してみよう
            

            :::message
**レッツトライ**
ターミナルで tmux と入力してセッションを開始
                Ctrl+B を押し、指を離して c を押す（新しいウィンドウができる）
                Ctrl+B を押し、指を離して , を押す（下のバーの色が変わり、名前入力待ちになる）
                好きな名前（例: editor）を入力してEnter
:::
          

          ### チャレンジ: プリフィックスキーの操作手順を書こう

tmux のプリフィックスキーを使った一連の操作手順を書いてください。セッション作成からウィンドウ作成、名前変更、デタッチまでの流れです。

```tsx
# プリフィックスキー操作の練習

# 1. tmux セッションを開始

# 2. 新しいウィンドウを作成
# (プリフィックス + ?)

# 3. ウィンドウに名前を付ける
# (プリフィックス + ?)

# 4. セッションからデタッチ
# (プリフィックス + ?)
```

:::details 模範解答
```tsx
# プリフィックスキー操作の練習

# 1. tmux セッションを開始
tmux

# 2. 新しいウィンドウを作成
# Ctrl+B C

# 3. ウィンドウに名前を付ける
# Ctrl+B ,

# 4. セッションからデタッチ
# Ctrl+B D
```
:::
