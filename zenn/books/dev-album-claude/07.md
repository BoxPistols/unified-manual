            ## 
              
              パーミッションルール
            
            
              `settings.json` でツール単位の許可/拒否ルールを定義できます。評価順序は **deny → ask → allow** です。
            

            ```json
// .claude/settings.json
{
  "permissions": {
    "allow": [
      "Bash(npm run *)",
      "Bash(git status)",
      "Bash(git diff *)",
      "Read"
    ],
    "ask": [
      "Bash(git push *)",
      "Bash(git commit *)"
    ],
    "deny": [
      "Bash(curl *)",
      "Read(./.env)",
      "Read(./secrets/**)"
    ]
  }
}
```

            ## 
              
              サンドボックス
            
            
              パーミッションとは別に、OS レベルのサンドボックスでプロセスを隔離できます。
            

            
              
                #### macOS
                Seatbelt（sandbox-exec）による分離

              

              
                #### Linux
                bubblewrap（bwrap）による分離

              

            

            :::message alert
**最小権限の原則**
パーミッションルールとサンドボックスは補完的なセキュリティレイヤーです。常に「最小権限の原則」を適用し、必要最小限の権限のみを付与してください。
:::

        ### チャレンジ: パーミッション設定を書いてみよう

.claude/settings.json に、npm/git の基本コマンドを許可し、.env ファイルの読み取りと curl の実行を拒否するパーミッションルールを書いてください。

```tsx
{
  "permissions": {
    "allow": [
      // npm と git の基本コマンドを許可
    ],
    "deny": [
      // .env の読み取りと curl を拒否
    ]
  }
}
```
