        # 
          JSX を理解する
        
        
          React のコードに登場する「HTML のようなもの」、それが JSX です。
          HTML との違いを知り、使いこなせるようになりましょう。
        

        :::message
**なぜ今これを学ぶのか**
React のコードを読み書きするうえで、JSX の理解は絶対に避けて通れません。
            見た目は HTML そっくりですが、実は JavaScript です。
            この違いを理解しないと、「なぜ class じゃなくて
            className なの？」
            という疑問が解消されません。
:::

        
          
          
            ## JSX とは何か？
            
              JSX は **JavaScript XML** の略で、JavaScript の中に HTML のような構文を書ける拡張構文です。
              React のコンポーネントが「見た目」を定義するために使います。
            

            ```tsx
// これが JSX
function Greeting() {
  return # こんにちは、React！
}

// JSX は実際にはこう変換される（知識として知っておくだけでOK）
function Greeting() {
  return React.createElement('h1', null, 'こんにちは、React！')
}
```

            
              `&lt;h1&gt;こんにちは&lt;/h1&gt;` と書くと、
              裏側では `React.createElement('h1', null, 'こんにちは')`
              に変換されます。つまり JSX は HTML ではなく、**JavaScript の関数呼び出しの糖衣構文（ショートカット）**です。
            

            
              :::message
**なぜ JSX を使うのか？**
React.createElement を毎回書くのは大変です。
                  JSX のおかげで、HTML を書く感覚で UI を定義できます。
                  デザイナーにとっても、JSX は HTML の知識を活かしやすい構文です。
:::
            

          

          
          
            ## JSX の裏側をもう少し深く
            
              JSX が `React.createElement` に変換されることは先ほど学びました。
              もう少し複雑な例で、この変換がどう行われるか見てみましょう。
              理解しておくと、JSX のルール（なぜルート要素が1つ必要か、など）が腑に落ちます。
            

            ```tsx
// JSX で書いた場合
function Card() {
  return (
    
      ## タイトル
      説明文です。

    

  )
}

// ↓ コンパイラ（Babel/SWC）がこう変換する ↓

function Card() {
  return React.createElement(
    'div',                          // 要素のタグ名
    { className: 'card' },          // 属性（props）
    React.createElement('h2', null, 'タイトル'),    // 子要素1
    React.createElement('p', null, '説明文です。')  // 子要素2
  )
}
```

            
              `React.createElement` は3つの引数を取ります。
            

            
              - **第1引数**: タグ名（'div'、'h2'）またはコンポーネント（Button）
              - **第2引数**: 属性のオブジェクト（className、style など）。なければ null
              - **第3引数以降**: 子要素（テキストや他の createElement 呼び出し）
            

            
              この構造を知っておくと、「JSX は必ず1つのルート要素で囲む必要がある」というルールが理解できます。
              `React.createElement` は1つの要素しか返せないため、
              複数の要素を返そうとするとエラーになるのです。
            

            
              :::message
**最新の React では React.createElement を意識しなくてよい**
React 17 以降は「新しい JSX トランスフォーム」が導入され、
                  import React from 'react' を書かなくても
                  JSX が使えるようになりました。内部的には jsx() という
                  別の関数が使われますが、仕組みの本質は同じです。
                  普段のコーディングでは意識する必要はありません。
:::
            

          

          
          
            ## HTML との違い
            
              JSX は HTML に似ていますが、いくつか重要な違いがあります。
              デザイナーが HTML を書いた経験があるなら、以下の点だけ気をつければ大丈夫です。
            

            
            ### 1. class → className
            
              HTML の `class` 属性は、
              JSX では `className` と書きます。
              JavaScript に `class` という予約語があるためです。
            

            ```tsx
// HTML
...

// JSX
...

```

            
            ### 2. for → htmlFor
            
              label タグの `for` 属性は
              `htmlFor` と書きます。
              こちらも JavaScript の予約語 `for` との衝突を避けるためです。
            

            ```tsx
// HTML
<label for="email">メールアドレス</label>

// JSX
<label htmlFor="email">メールアドレス</label>

```

            
            ### 3. ケバブケース → キャメルケース
            
              HTML ではハイフン区切り（ケバブケース）で書く属性名が、JSX ではキャメルケースになります。
            

            ```tsx
// HTML

// JSX

```

            
            ### 4. 自己閉じタグは必須
            
              HTML では `&lt;img&gt;` や
              `&lt;br&gt;` は閉じタグなしで書けますが、
              JSX では必ず閉じる必要があります。
            

            ```tsx
// HTML（閉じなくてもOK）

// JSX（必ず閉じる）

```

            
              :::message alert
**よくあるエラー**
class と書いてしまうのが初心者が最もよくやるミスです。
                  エラーにはならず警告が出るだけなので気づきにくいですが、必ず
                  className を使いましょう。
                  VS Code の ESLint 拡張を入れていれば、自動で指摘してくれます。
:::
            

          

          
          
            ## よくある JSX エラーと対処法
            
              JSX を書き始めると、必ず遭遇するエラーがいくつかあります。
              ここで先に知っておけば、エラーに出会っても慌てずに対処できます。
            

            
            ### エラー1: 複数のルート要素
            
              JSX は必ず1つのルート要素で囲む必要があります。2つ以上の要素をそのまま返すとエラーになります。
            

            ```tsx
// NG: コンパイルエラーが発生！

  return (
    # タイトル
    本文

  )
}
// エラー: JSX expressions must have one parent element.

// OK: div で囲む

  return (
    
      # タイトル
      本文

    

  )
}

// ベスト: フラグメント（<> </>）で囲む（余計な DOM が増えない）
function App() {
  return (
    <>
      # タイトル
      本文

    </>
  )
}
```

            
            ### エラー2: JSX の中で if 文を使おうとする
            
              JSX の波括弧 `{'{ }'}` の中には「式」しか書けません。
              `if` や `for` は
              「文」なので直接書くことができません。
            

            ```tsx
// NG: JSX の中に if 文は書けない！
function Greeting({ isLoggedIn }) {
  return (
    
      {if (isLoggedIn) { return おかえりなさい
 }}
    

  )
}
// エラー: Unexpected token

// OK: 三項演算子を使う
function Greeting({ isLoggedIn }) {
  return (
    
      
    

  )
}

// OK: JSX の外側で if 文を使う
function Greeting({ isLoggedIn }) {
  let message
  if (isLoggedIn) {
    message = おかえりなさい

  } else {
    message = ログインしてください

  }

  return 

}
```

            :::message
**「式」と「文」の違い（覚え方）**
式（Expression）: 値を返すもの。変数に代入できるもの。
                例: 1 + 2、
                isLoggedIn ? "はい" : "いいえ"、
                items.map(...)
                
                文（Statement）: 処理の手順を記述するもの。値を返さない。
                例: if (...) {'{ }'}、
                for (...) {'{ }'}、
                switch (...) {'{ }'}
                
                覚え方: 「const x = ___ の ___ に入れられるなら式」
:::

            ## {'フラグメント（<> </>）'}
            
              JSX には「1つのルート要素で囲む」というルールがあります。
              複数の要素を並べて返したい場合、余計な `&lt;div&gt;` で囲むと
              不要な DOM 要素が増えてしまいます。そこで使うのが**フラグメント**です。
            

            ```tsx
// NG: 複数の要素をそのまま返せない
function App() {
  return (
    # タイトル
    本文

  )
  // エラー: JSX expressions must have one parent element
}

// OK: div で囲む（でも余計な div が増える）
function App() {
  return (
    
      # タイトル
      本文

    

  )
}

// ベスト: フラグメントを使う（DOMに余計な要素が追加されない）
function App() {
  return (
    <>
      # タイトル
      本文

    </>
  )
}
```

            
              `&lt;&gt; &lt;/&gt;` は
              `&lt;React.Fragment&gt;` の省略形です。
              実際の DOM にはフラグメントは残らないので、レイアウトに影響を与えません。
            

            
              :::message
**div とフラグメント、どっちを使う？**
迷ったら、まずフラグメント &lt;&gt; &lt;/&gt; を使いましょう。
                  スタイリングのために囲む要素が必要な場合（flexbox のコンテナなど）は
                  &lt;div&gt; を使います。
                  「この div は見た目のために必要か？」を判断基準にするとよいでしょう。
:::

            ## インラインスタイルの書き方
            
              JSX でインラインスタイルを指定する場合は、文字列ではなく JavaScript オブジェクトで渡します。
              プロパティ名はキャメルケースで書きます。
            

            ```tsx
// HTML のインラインスタイル
// 

// JSX のインラインスタイル

  スタイル付きの要素

```

            
              二重の波括弧 `{'{{}}'}` に見えますが、
              外側は「JSX の式埋め込み」、内側は「JavaScript オブジェクトのリテラル」です。
