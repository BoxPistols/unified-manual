              ### プラグインが行う最適化
              
                
                  **1. ソースマッピング**
                  開発者ツールで、生成されたクラス名がどのコンポーネント・ファイルに由来するかを確認できるようになります。

                

                
                  **2. ラベル付け**
                  生成されるクラス名に変数名やコンポーネント名が含まれ、デバッグしやすくなります（例: `css-cardStyle-1a2b3c`）。

                

                
                  **3. CSS の最適化**
                  不要な空白や改行を除去し、CSS 文字列を圧縮します。

                

                
                  **4. 静的スタイルの事前評価**
                  props に依存しない静的なスタイルをビルド時に事前計算し、ランタイムのオーバーヘッドを削減します。

            ## まとめ

            
              
                - 
                  1.

                  **Emotion** は css prop と styled の2つのアプローチを提供する CSS-in-JS ライブラリ

                
                - 
                  2.

                  **css prop** は新しいコンポーネントを作らずに既存要素にスタイルを適用でき、手軽さが魅力

                
                - 
                  3.

                  **スタイルの合成**（配列での結合）は Emotion の大きな強み

                
                - 
                  4.

                  **TypeScript** でテーマ型を拡張すると、テーマ値の型チェック・補完が効くようになる

                
                - 
                  5.

                  **MUI v5** が Emotion をデフォルトのスタイリングエンジンとして採用しているため、MUI と組み合わせる場合に特に有力
