        # React 19 の新しい Hooks
        
          React 19 では、フォーム処理や非同期操作をよりシンプルに扱うための新しい Hooks が追加されました。
          useActionState、useOptimistic、useFormStatus を使うことで、
          これまで手動で管理していた「送信中」「エラー」「楽観的更新」の状態管理が劇的に簡単になります。
        

        :::message
**なぜ今これを学ぶのか**
これまでの React では、フォーム送信時の loading 状態やエラーハンドリングを
            useState と useEffect を組み合わせて手動で管理する必要がありました。
            React 19 の新しい Hooks は、こうした「よくあるパターン」を公式にサポートします。
            特に Server Actions と組み合わせたとき、その真価を発揮します。
:::

              useActionState なら 1 つの Hook で完結します。

**useFormStatus は子コンポーネントから呼ぶ**
useFormStatus は &lt;form&gt; を直接レンダリングしているコンポーネントでは使えません。

            ## useTransition の改善点: async transitions
            
              React 18 で追加された `useTransition` が
              React 19 でさらに強化されました。最大の変更点は、
              **startTransition に非同期関数（async function）を渡せる**ようになったことです。

            ### useTransition vs useActionState の使い分け
            
              
                - 
                  **useActionState**: フォームの送信処理に最適。
                  `&lt;form action=&#123;...&#125;&gt;` と連携し、FormData を自動的に受け取る。
                  結果の状態管理もセットになっている。
                
                - 
                  **useTransition**: フォーム以外の非同期操作（検索、フィルタ、ページ遷移など）に最適。
                  既存の state 更新を「優先度の低い更新」としてマークする。
                  isPending で処理中かどうかだけを追跡する。

            ## まとめ
            
              
                1

                
                  **useActionState**: フォームアクションの結果（成功・失敗）と isPending を 1 つの Hook で管理
                

              

              
                2

                
                  **useOptimistic**: サーバー応答を待たずに UI を即座に更新。失敗時は自動で元に戻る
                

              

              
                3

                
                  **useFormStatus**: 子コンポーネントから親フォームの送信状態を取得。再利用可能なボタンに最適
                

              

              
                4

                
                  **useTransition**: React 19 で async 対応に。非同期操作中も isPending で UX をスムーズに
