        # 
          フォーム入門
        
        
          ログインフォーム、問い合わせフォーム、設定画面 ── Web アプリに欠かせないフォームの実装パターンを学びます。React では「制御コンポーネント」というアプローチで、フォームの状態を完全にコントロールします。
        

        :::message
**なぜ今これを学ぶのか**
useState、イベントハンドリング、条件分岐 ── ここまで学んだ知識が、フォーム実装ですべてつながります。フォームは Web アプリの中で最もインタラクティブな部分であり、これらの知識を総動員する実践的なテーマです。
          
          
            デザイナーとして「フォームの入力状態」「バリデーションエラー」「送信中の表示」をデザインしてきた方も多いでしょう。React でそれらを実装する方法を理解することで、デザインと実装の橋渡しができるようになります。
:::

**なぜ制御コンポーネントを使うのか？**
value と onChange を使うことで、React が入力値の「唯一の真実の情報源（Single Source of Truth）」になります。バリデーション、値のフォーマット、条件付きの入力制限など、あらゆる制御が可能になります。
:::
          

          
          
            ## 非制御コンポーネント（Uncontrolled Components）
            
              制御コンポーネントの対となる方法が「非制御コンポーネント」です。state で値を管理せず、DOM 自体が値を保持します。`useRef` を使って DOM から直接値を取得します。
            

            ```tsx
import { useRef } from 'react';

function UncontrolledInput() {
  // useRef で DOM 要素への参照を作成
  const inputRef = useRef<HTMLInputElement>(null);

  const handleSubmit = (e: React.FormEvent) => {
    e.preventDefault();
    // DOM から直接値を取得
    const value = inputRef.current?.value;
    console.log('入力値:', value);
  };

  return (

        送信

  );
}

// - ファイル入力（<input type="file">）
// - サードパーティライブラリとの連携
// - 送信時にだけ値が必要で、リアルタイムの制御が不要な場合
```

            ## 1つのハンドラで複数の入力を管理する
            
              入力フィールドが増えるたびに個別の onChange を書くのは面倒です。`name` 属性を活用して、1つのハンドラで全フィールドを管理するテクニックがあります。
            

            ```tsx
import { useState } from 'react';

interface ProfileFormData {
  firstName: string;
  lastName: string;
  email: string;
  phone: string;
  company: string;
  role: string;
}

function ProfileForm() {
  const [formData, setFormData] = useState<ProfileFormData>({
    firstName: '',
    lastName: '',
    email: '',
    phone: '',
    company: '',
    role: '',
  });

  // 汎用的な変更ハンドラ
  // name 属性を使って、どのフィールドが変更されたかを判別
  const handleChange = (
    e: React.ChangeEvent<HTMLInputElement | HTMLSelectElement>
  ) => {
    const { name, value } = e.target;
    setFormData((prev) => ({
      ...prev,
      [name]: value,  // Computed Property で動的にキーを指定
    }));
  };

  const handleSubmit = (e: React.FormEvent) => {
    e.preventDefault();
    console.log(formData);
  };

  return (

          <label >姓</label>

          <label >名</label>

        <label >メール</label>

        <label >電話番号</label>

        <label >会社名</label>

        <label >役職</label>

          <option value="designer">デザイナー</option>
          <option value="engineer">エンジニア</option>
          <option value="pm">プロダクトマネージャー</option>
          <option value="other">その他</option>

        保存

}
```
            :::message
**このパターンのメリット**
フィールドが10個あっても handleChange は1つで済みます。ポイントは input の name 属性を state のキー名と一致させることです。[name]: value という Computed Property Names の構文で、動的にオブジェクトのキーを指定しています。
:::
          

          
          
            ## フォームバリデーションの基本
            
              ユーザーが正しい形式でデータを入力しているかをチェックするのがバリデーションです。リアルタイムバリデーションと送信時バリデーションの2つの方法があります。
            

            ### リアルタイムバリデーション vs 送信時バリデーション
            
              
                #### リアルタイムバリデーション
                
                  - - 入力中またはフォーカスが外れた時に検証
                  - - ユーザーにすぐフィードバックを返せる
                  - - パスワード強度表示などに最適
                  - - 過度に使うと煩わしい
                

              

              
                #### 送信時バリデーション
                
                  - - 送信ボタン押下時に一括検証
                  - - 入力中にエラーが出ないので快適
                  - - シンプルなフォームに最適
                  - - エラー箇所が一目でわかるようにする
                

              

            

            ```tsx
import { useState } from 'react';

interface FormErrors {
  name?: string;
  email?: string;
  message?: string;
}

function ValidatedForm() {
  const [name, setName] = useState('');
  const [email, setEmail] = useState('');
  const [message, setMessage] = useState('');
  const [errors, setErrors] = useState<FormErrors>({});
  const [touched, setTouched] = useState<Record<string, boolean>>({});

  // バリデーション関数
  const validate = (): FormErrors => {
    const newErrors: FormErrors = {};

    if (!name.trim()) {
      newErrors.name = 'お名前は必須です';
    } else if (name.length < 2) {
      newErrors.name = 'お名前は2文字以上で入力してください';
    }

    if (!email.trim()) {
      newErrors.email = 'メールアドレスは必須です';
    } else if (!/^[^\s@]+@[^\s@]+\.[^\s@]+$/.test(email)) {
      newErrors.email = '正しいメールアドレスの形式で入力してください';
    }

    if (!message.trim()) {
      newErrors.message = 'メッセージは必須です';
    } else if (message.length < 10) {
      newErrors.message = 'メッセージは10文字以上で入力してください';
    }

    return newErrors;
  };

  // フォーカスが外れたとき（blur）にバリデーション
  const handleBlur = (field: string) => {
    setTouched((prev) => ({ ...prev, [field]: true }));
    setErrors(validate());
  };

  // 送信時にバリデーション
  const handleSubmit = (e: React.FormEvent) => {
    e.preventDefault();

    // 全フィールドを touched にする
    setTouched({ name: true, email: true, message: true });

    const validationErrors = validate();
    setErrors(validationErrors);

    // エラーがなければ送信
    if (Object.keys(validationErrors).length === 0) {
      console.log('送信:', { name, email, message });
      alert('送信しました！');
    }
  };

  return (

      ## お問い合わせ

          お名前 *

        {touched.name && errors.name && (
          

        )}

          メールアドレス *

        {touched.email && errors.email && (
          

        )}

          メッセージ *

          /500

        {touched.message && errors.message && (
          

        )}

        送信する

  );
}
```
            :::message
**touched の役割**
touched はフィールドが一度でもフォーカスされたかを記録します。これにより、まだ触れていないフィールドにエラーメッセージが表示されるのを防ぎます。ユーザーがフィールドに入力してフォーカスを外したとき（blur）に初めてバリデーションエラーを表示する、というのが一般的な UX パターンです。
:::

            ## 実践パターン: フォームの状態管理
            
              本格的なフォームでは、入力データだけでなく送信状態（idle / submitting / success / error）も管理する必要があります。以下のパターンを押さえておきましょう。
            

            ```tsx
import { useState } from 'react';

type FormStatus = 'idle' | 'submitting' | 'success' | 'error';

function PracticalForm() {
  const [formData, setFormData] = useState({
    name: '', email: '', subject: '', message: '',
    newsletter: false, privacyPolicy: false,
  });
  const [status, setStatus] = useState<FormStatus>('idle');
  const [errors, setErrors] = useState<Record<string, string>>({});

  // 汎用 handleChange（テキスト + チェックボックス対応）
  const handleChange = (
    e: React.ChangeEvent<HTMLInputElement | HTMLSelectElement | HTMLTextAreaElement>
  ) => {
    const { name, type } = e.target;
    const value = type === 'checkbox'
      ? (e.target as HTMLInputElement).checked
      : e.target.value;
    setFormData((prev) => ({ ...prev, [name]: value }));
    if (errors[name]) setErrors((prev) => ({ ...prev, [name]: '' }));
  };

  const handleSubmit = async (e: React.FormEvent) => {
    e.preventDefault();
    // バリデーション → 送信 → 状態管理
    const newErrors: Record<string, string> = {};
    if (!formData.name.trim()) newErrors.name = '必須項目です';
    if (!formData.email.trim()) newErrors.email = '必須項目です';
    if (!formData.privacyPolicy) newErrors.privacyPolicy = '同意が必要です';
    if (Object.keys(newErrors).length > 0) { setErrors(newErrors); return; }

    setStatus('submitting');
    try {
      await new Promise((r) => setTimeout(r, 2000)); // API 呼び出し
      setStatus('success');
    } catch {
      setStatus('error');
    }
  };

  // 送信完了画面
  if (status === 'success') {
    return (

        ありがとうございます。

      

    );
  }

  return (

          送信に失敗しました。再度お試しください。
        

      )}

      {errors.name && 
}

        {status === 'submitting' ? '送信中...' : '送信する'}
```

**実践フォームのポイント**
状態管理: formData（入力値）、errors（エラー）、status（送信状態）の3つを分けて管理
                汎用ハンドラ: name 属性 + Computed Property で1つの handleChange で全フィールドを管理
                UX: 送信中はボタンを無効化し、完了画面・エラー表示を条件分岐で切り替える

            ## まとめ
            
              
                ### フォームの基本
                
                  - - value + onChange = 制御コンポーネント
                  - - ref + defaultValue = 非制御コンポーネント
                  - - onSubmit + preventDefault でフォーム送信
                  - - name 属性で汎用ハンドラを実現
                  - - チェックボックスは checked + onChange
                

              

              
                ### UX のポイント
                
                  - - blur 時にバリデーションを実行する
                  - - 送信中は loading 表示 + ボタン無効化
                  - - エラー/成功状態を明確に伝える
                  - - 必須項目は * で示す
                  - - 複雑なフォームにはライブラリを検討
