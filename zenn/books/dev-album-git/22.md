          # 
            開発環境の統合確認
          
          
            すべてのツールが正しくインストールされたか確認し、初めてのプロジェクトを作成します。
          

        

      


      
      
        
        
          ## 
            インストール確認
          
          
            
              以下のコマンドを実行して、すべてのツールが正しくインストールされたか確認しましょう。
            


            
              
              
                ### Git のバージョン確認
                ```bash
git --version
```
                
                  実行結果例：`git version 2.40.0`
                

              


              
              
                ### Node.js のバージョン確認
                ```bash
node --version
npm --version
```
                
                  実行結果例：`v18.17.0` と `9.6.7`
                

              


              
              
                ### Cursor の確認
                
                  Cursor がアプリケーションフォルダに存在し、起動できることを確認してください。
                

              


              
              
                ### VSCode の確認
                
                  VSCode がアプリケーションフォルダに存在し、起動できることを確認してください。
                

                ```bash
code --version
```
              

            


            :::message
すべてのコマンドが実行でき、バージョン番号が表示されれば、インストールは成功です！
:::
          

        


        
        
          ## 
            初めてのプロジェクト作成
          

          
          
            
              
                1
              

              ### 
                プロジェクトフォルダを作成
              
            

            
              ターミナルで以下のコマンドを実行して、プロジェクトフォルダを作成します。
            

            ```bash
mkdir my-first-project
cd my-first-project
```
          


          
          
            
              
                2
              

              ### 
                Git リポジトリを初期化
              
            

            
              プロジェクトフォルダで Git を初期化します。
            

            ```bash
git init
```
            
              実行結果に「Initialized empty Git repository」と表示されれば成功です。
            

          


          
          
            
              
                3
              

              ### 
                最初のファイルを作成
              
            

            
              README.md ファイルを作成して、プロジェクトの説明を書きます。
            

            ```bash
touch README.md
echo "# My First Project" > README.md
```
          


          
          
            
              
                4
              

              ### 
                VSCode で開く
              
            

            
              VSCode でプロジェクトを開きます。
            

            ```bash
code .
```
            
              VSCode が起動し、プロジェクトフォルダが表示されます。
            

          


          
          
            
              
                5
              

              ### 
                Git で管理する
              
            

            
              VSCode の統合ターミナルで、ファイルを Git に追加して Commit します。
            

            ```bash
git add README.md
git commit -m "Initial commit: Add README"
```
            :::message
これで、初めてのプロジェクトが Git で管理されました！
:::
          

        


        
        
          ## 
            実践的な開発フロー
          
          
            
              これからの開発は、以下のフローで進めていきます。
            


            
              
                
                  1
                

                
                  ### ファイルを編集
                  VSCode でコードを書きます。

                

              


              
                
                  2
                

                
                  ### 変更を確認
                  
                    `git status` でファイルの変更を確認します。
                  

                

              


              
                
                  3
                

                
                  ### ステージング
                  
                    `git add .` で変更をステージングします。
                  

                

              


              
                
                  4
                

                
                  ### Commit
                  
                    `git commit -m "メッセージ"` で変更を記録します。
                  

                

              


              
                
                  5
                

                
                  ### Push（GitHub に保存）
                  
                    `git push` で GitHub に送信します。
                  

                

              

            


            :::message
このフローを繰り返すことで、チーム開発や大規模プロジェクトに対応できるようになります。
:::

          ### チャレンジ: プロジェクト作成から Git 管理までの流れを書いてみよう

フォルダ作成、Git 初期化、ファイル作成、コミットまでの一連のコマンドを書いてください。

```tsx
# 1. プロジェクトフォルダを作成して移動

# 2. Git リポジトリを初期化

# 3. README.md を作成して内容を書き込む

# 4. ファイルをステージングしてコミット

```

:::details 模範解答
```tsx
# 1. プロジェクトフォルダを作成して移動
mkdir my-first-project
cd my-first-project

# 2. Git リポジトリを初期化
git init

# 3. README.md を作成して内容を書き込む
touch README.md
echo "# My First Project" > README.md

# 4. ファイルをステージングしてコミット
git add README.md
git commit -m "Initial commit: Add README"
```
:::
