          # 
            Markdown を覚えよう
          
          
            開発者・AI との共通言語「Markdown」を身につけましょう。覚えることは少ないのに、効果は絶大です。
          

        

      


      
      
        :::message
**なぜ今これを学ぶのか**
Markdown は、記号を使ってテキストに見出しや太字・リストを付ける記法です。GitHub の README やコメント欄で使われています。
          
          
            Word で太字にするには Ctrl+B を押しますが、Markdown では **太字** と書きます。HTML タグより短く、プレーンテキストとして読んでも意味が通ります。Notion や Obsidian でも同じ記法が使えます。
          
          
            ChatGPT や Claude の回答も Markdown で書かれているため、基本的な記法を知っておくと読みやすくなります。
:::

        
        
          ## 
            なぜ Markdown を覚えるべきなのか
          
          
            
              リポジトリを作成したとき、最初に目にするファイルが **README.md** です。この「.md」がまさに Markdown ファイルを示す拡張子です。
            

            
              Markdown は**テキストに簡単な記号を付けるだけで、見出し・リスト・リンク・コードブロックなどを表現できる軽量な書式ルール**です。HTML のように複雑なタグを書く必要はなく、覚えるルールもごくわずかです。
            


            
              ### Markdown を覚えるメリット
              
                - 
                  ✓

                  **AI との対話がスムーズになる** ─ ChatGPT や Claude は Markdown で回答します。あなたも Markdown で指示を出すと、正確に意図が伝わります

                
                - 
                  ✓

                  **GitHub で必須** ─ README、Issue、Pull Request、コメントすべてが Markdown で書かれています

                
                - 
                  ✓

                  **覚えることが極めて少ない** ─ 基本記法は 10 個程度。15 分で一通り使えるようになります

                
                - 
                  ✓

                  **どこでも使える** ─ Notion、Slack、Qiita、Zenn、技術ブログなど、多くのサービスが対応しています

                
              

            


            :::message
**AI 時代の共通言語**
AI ツール（ChatGPT、Claude、GitHub Copilot など）は Markdown 形式でやり取りするのが標準です。「見出しで構造を示す」「リストで箇条書きにする」「コードブロックでコードを囲む」─ これだけで AI への指示精度が大きく向上します。Markdown は人間と AI の共通言語と言えます。
:::

          ## 
            基本の書き方
          

          
            
            
              
                
                  1
                

                ### 
                  見出し（Headings）
                
              

              
                「#」の数で見出しのレベルを指定します。# が少ないほど大きい見出しです。
              


              
                
                  Markdown（入力）

                  ```markdown
# 見出し 1（最大）
## 見出し 2
### 見出し 3
#### 見出し 4
```
                

                
                  表示結果（イメージ）

                  
                    見出し 1（最大）

                    見出し 2

                    見出し 3

                    見出し 4

                  

                

              

            


            
            
              
                
                  2
                

                ### 
                  テキスト装飾（太字・斜体・取り消し線）
                
              


              
                
                  Markdown（入力）

                  ```markdown
**太字（ボールド）**
*斜体（イタリック）*
~~取り消し線~~
***太字かつ斜体***
```
                

                
                  表示結果（イメージ）

                  
                    **太字（ボールド）**

                    *斜体（イタリック）*

                    <s>取り消し線</s>

                    ***太字かつ斜体***

                  

                

              

            


            
            
              
                
                  3
                

                ### 
                  リスト（箇条書き・番号付き）
                
              


              
                
                  Markdown（入力）

                  ```markdown
- 項目 A
- 項目 B
  - サブ項目 B-1
  - サブ項目 B-2
- 項目 C
```
                

                
                  表示結果（イメージ）

                  
                    
                      - 項目 A
                      - 項目 B
                        
                          <li>サブ項目 B-1
                          - サブ項目 B-2
                        

                      </li>
                      - 項目 C
                    

                  

                

              


              
                
                  ```markdown
1. 手順 1
2. 手順 2
3. 手順 3
```
                

                
                  
                    
                      - 手順 1
                      - 手順 2
                      - 手順 3
                    

                  

                

              

            


            
            
              
                
                  4
                

                ### 
                  リンクと画像
                
              


              ```markdown
[表示テキスト](https://example.com)

![代替テキスト](画像のURL)
```

              
                リンクは `[テキスト](URL)`、画像は先頭に `!` を付けるだけです。
              


              :::message
GitHub の Issue やコメントでは、画像をドラッグ＆ドロップするだけで自動的に Markdown の画像記法に変換してくれます。
:::
            


            
            
              
                
                  5
                

                ### 
                  コード（インラインとブロック）
                
              

              
                コードを共有するときに最も重要な記法です。AI にコードを見せるときも、必ずコードブロックで囲みましょう。
              


              
                
                  インラインコード（文中に埋め込む）

                  
                  
                    `git status` コマンドを実行してください

                  

                


                
                  コードブロック（複数行のコード）

                  
                  
                    バッククォート3つ（```）で囲み、言語名を指定するとシンタックスハイライト（色付け）されます。
                  

                

              


              :::message
**AI への質問時のコツ**
AI にコードについて質問するときは、必ずコードブロック（```）で囲んで渡しましょう。AI がコードの構造を正しく理解でき、より的確な回答が得られます。
:::
            


            
            
              
                
                  6
                

                ### 
                  引用（Blockquote）
                
              


              
                
                  ```markdown
> これは引用文です。
> 他の人の言葉を引用するときに使います。
```
                

                
                  表示結果（イメージ）

                  
                    <blockquote >
                      これは引用文です。他の人の言葉を引用するときに使います。
                    </blockquote>
                  

                

              

            


            
            
              
                
                  7
                

                ### 
                  水平線（区切り線）
                
              


              ```markdown
セクション 1 の内容

---

セクション 2 の内容
```
              
                `---` または `***` で水平線を引けます。セクションの区切りに便利です。

**チェックボックスはプラットフォームによって対応が異なります**

:::

**GitHub Flavored Markdown（GFM）とは**
GitHub が標準の Markdown を独自に拡張したものです。テーブル、チェックボックス、取り消し線、自動リンクなどの便利な機能が追加されています。GitHub 上では GFM が標準なので、このマニュアルで紹介した記法はすべて GitHub で使えます。
:::

            
              ### 覚えておきたいポイント
              
                - 
                  1

                  **基本記法（見出し・リスト・太字・リンク・コードブロック）はほぼどこでも使える**ので、迷わず使ってOK

                
                - 
                  2

                  **テーブルやチェックボックスは拡張記法**なので、環境によっては表示が崩れることがある

                
                - 
                  3

                  **Slack は独自の記法**を持っており、標準の Markdown とは書き方が異なる部分がある（例：太字は `*テキスト*` ではなく独自の方式）

                
              

            

          

        


        
        
          ## 
            実践：README.md を書いてみよう
          
          
            
              前のステップで作成したリポジトリの README.md を、Markdown を使って書き換えてみましょう。以下は README のテンプレートです。
            


            ```markdown
# My First Project

GitHub と Git の学習用リポジトリです。

## このリポジトリについて

このリポジトリでは、以下の内容を学習しています：

- Git の基本操作（add, commit, push）
- GitHub でのリポジトリ管理
- Markdown の書き方

## 使い方

\`\`\`bash
git clone git@github.com:your-username/my-first-project.git
cd my-first-project
\`\`\`

## 学習の進捗

- [x] GitHub アカウント作成
- [x] リポジトリ作成
- [x] Markdown の基本を学習
- [ ] 最初の Commit
- [ ] Push を実行

## メモ

> Markdown は覚えることが少ないのに、使える場所がとても多い便利な記法です。
```

            :::message
**README.md のコツ**
README は「プロジェクトの顔」です。何のプロジェクトか、どう使うか、を簡潔に書くことを意識しましょう。最初は完璧でなくて大丈夫です。後からいくらでも更新できます。
:::

          ### チャレンジ: Markdown で自己紹介を書いてみよう

以下の要素を含む Markdown を書いてください：見出し（# または ##）、箇条書きリスト（- で始まる項目を2つ以上）、太字（**テキスト**）。内容は自由です。

```tsx
# 

- 
- 
```

:::details 模範解答
```tsx
# 自己紹介

**名前**: テスト太郎

- Git を学習中
- Markdown が書けるようになった
```
:::
