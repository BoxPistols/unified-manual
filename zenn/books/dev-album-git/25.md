            ### リポジトリに移動して Claude Code を起動する
          

          
            Claude Code は「今いるフォルダ」を作業対象として認識します。
            まず、操作したいリポジトリのフォルダに移動してから起動しましょう。
          


          {selectedOS === 'mac' ? (
            ```bash
# 例: 自分のリポジトリフォルダに移動
cd ~/projects/my-repo

# Claude Code を起動
claude
```
          ) : (
            ```powershell
# 例: 自分のリポジトリフォルダに移動
cd C:\Users\YourName\projects\my-repo

# Claude Code を起動
claude
```
          )}

          :::message
**フォルダの場所がわからない場合**
Cursor や VSCode でプロジェクトを開いている場合は、エディタ内のターミナルを使うと
            自動的にプロジェクトフォルダにいる状態になっています。そのまま claude と入力するだけでOKです。
:::

          
            Claude Code が認識する範囲

            
              Claude Code は起動時のフォルダ以下のファイルを読み取れます。
              Git リポジトリのルート（`.git` フォルダがある場所）で起動するのがベストです。
            

          

        


        
        
          
            
              2
            

            ### リポジトリの状態を確認してもらう
          

          
            Claude Code にリポジトリの状態を確認してもらいましょう。
            自然な言葉で頼むだけでOKです。
          


          ```text
# リポジトリの状態を確認
> このリポジトリの状態を教えて

# Claude が以下のような情報を教えてくれます:
# - 現在のブランチ名
# - 未コミットの変更があるか
# - リモートとの差分があるか
```

          ```text
# もっと詳しく聞いてみましょう
> このプロジェクトの構成を教えて。どんなファイルがある？
```

          :::message
Claude Code は git status、git log などのコマンドを自動的に実行して、
            結果をわかりやすく説明してくれます。コマンドを覚える必要はありません。
:::
        


        
        
          
            
              3
            

            ### 最新の状態を取得する（fetch / pull）
          

          
            チームメンバーやリモートで変更があった場合、それを自分のローカルに取り込む必要があります。
            Claude Code に頼むだけで、fetch と pull を実行してくれます。
          


          ```text
# リモートの最新情報を確認（fetch）
> リモートに新しい変更があるか確認して
```

          ```text
# 最新の変更を取り込む（pull）
> リモートの最新の変更を取り込んで（pull して）
```

          
            fetch と pull の違い（おさらい）

            
              
                
                  
                  git fetch
                

                
                  リモートの情報を「確認」するだけ。自分のファイルは変わらない。安全な操作。
                

              

              
                
                  
                  git pull
                

                
                  リモートの変更を「取り込む」。自分のファイルが更新される。
                

              

            

          


          :::message alert
**pull するタイミング**
pull する前に、自分の未コミットの変更がないか確認しましょう。
            Claude Code に「pull しても大丈夫？」と聞くと、
            状態を確認してからアドバイスしてくれます。
:::

            ### 画面で確認する
          

          
            pull で最新の状態を取り込んだら、実際にブラウザやエディタで確認してみましょう。
            Web プロジェクトの場合は開発サーバーを起動して確認できます。
          


          ```text
# 開発サーバーの起動を Claude Code に頼む
> 開発サーバーを起動して

# Claude が package.json を確認して、適切なコマンドを実行してくれます
# 例: npm run dev, pnpm dev など
```

          :::message
**ブラウザで確認**
開発サーバーが起動したら、表示される URL（通常は http://localhost:5173 など）を
            ブラウザで開いて確認します。
:::

          
            もちろん、Claude Code を使わずに自分でコマンドを実行してもOKです。
            Claude Code はあくまでアシスタントなので、慣れてきたら自分で直接コマンドを打つ場面も出てきます。
          


          {selectedOS === 'mac' ? (
            ```bash
# 自分で直接実行する場合の例
npm run dev

# または
pnpm dev
```
          ) : (
            ```powershell
# 自分で直接実行する場合の例
npm run dev

# または
pnpm dev
```
          )}

            覚えておくと便利な Claude Code の使い方をまとめます。

            Claude Code の基本コマンド

            
              `claude` — Claude Code を起動

              `/exit` — Claude Code を終了

              `/help` — ヘルプを表示

              `/clear` — 会話履歴をクリア

              `Ctrl + C` — 処理を中断 / 終了

            

          

        


        
        
          ## 
            実践チャレンジ
          
          
            
              以下の操作を Claude Code を使って実際にやってみましょう:
            

            
              - 自分のリポジトリフォルダで `claude` を起動する
              - 「このリポジトリのブランチ一覧を教えて」と聞く
              - 「リモートの最新を pull して」と頼む
              - 「開発サーバーを起動して」と頼む
              - ブラウザで画面を確認する
              - `/exit` で Claude Code を終了する
            

          

        


        
        
          ### チャレンジ: CLAUDE.md を書いてみよう

Claude Code がプロジェクトのルールを認識するための CLAUDE.md ファイルを書いてください。プロジェクト名、使用言語、コミットメッセージのルールを含めましょう。

```tsx
# プロジェクト名

## 基本方針


```

:::details 模範解答
```tsx
# My Project

## 基本方針

- 言語: 日本語でコミュニケーション
- TypeScript を使用

## Git ワークフロー

- コミットメッセージは日本語で簡潔に
- push 前に必ずビルド確認
```
:::
